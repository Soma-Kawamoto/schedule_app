# Git 追跡対象の完全除外ガイド

一度 Git の管理下（コミット対象）に入ってしまったファイルは、`.gitignore` に追記しただけでは無視されません。以下の手順で「インデックス（キャッシュ）」から削除する必要があります。

## 📋 基本の 3 ステップ

### 1. .gitignore にファイル/フォルダを追記する
まず、今後二度と追跡されないように設定ファイルに書き込みます。

```bash
echo "path/to/filename" >> .gitignore
# フォルダ全体を対象にする場合
echo "folder_name/" >> .gitignore
```

### 2. Git のキャッシュ（インデックス）から削除する
ここが最重要です。 ファイルの実体は消さずに、Git の管理名簿からだけ名前を消します。
```bash
# 特定のファイルを外す
git rm --cached path/to/filename
# フォルダ全体を外す
git rm -r --cached folder_name/
```
### 3. 変更を確定させて Push する
「管理を辞めたこと」をコミットして、リモートリポジトリ（GitHub/GitLab）に反映させます。
```bash
git add .gitignore
git commit -m "Stop tracking and ignore large/unnecessary files"
git push
```

## 🛠 Gitコマンド・オプション役割一覧

| コマンド / オプション | 役割 | 補足・注意点 |
| :--- | :--- | :--- |
| `git rm --cached <file>` | **追跡解除（ファイル維持）** | Gitの管理名簿（インデックス）からのみ削除。PC上のファイルは消えません。 |
| `git rm <file>` | **完全削除** | Gitの管理からも、PC上のフォルダからもファイルを完全に削除します。 |
| `-r` (recursive) | **再帰処理** | フォルダを指定する際に必須。中身のファイルまでまとめて処理します。 |
| `git reset --soft HEAD^` | **直前のコミット取消** | 「コミットした」という事実だけ消し、編集内容はステージング状態で残ります。 |
| `git status` | **状態確認** | どのファイルが追跡対象外か、キャッシュが消えたかを確認する最重要コマンド。 |

### ⚠️ 注意：過去の履歴について
上記の手順は 「これからのコミット」 にそのファイルを含めないための方法です。

もし、過去のコミットに含まれている巨大なファイル が原因で git push がエラーになる場合は、以下のコマンドでコミット履歴を遡って修正する必要があります。

```Bash
# 直前のコミットを取り消してやり直す場合
git reset --soft HEAD^
```

### まとめ
- 書く (`.gitignore` に追記)
- 消す (`git rm --cached`)
- 送る (`commit & push`)
この流れをセットで覚えておけば、データベースファイルや機密情報（APIキーなど）を誤ってアップロードし続けることを防げます。
